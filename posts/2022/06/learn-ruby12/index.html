學習 Ruby 123 - 12 Block

June 13, 2022

用 do .. end 與 { }#

這兩個結果是相等的

# 使用 do .. end
5.times do |i|
 puts i
end

# 使用 {}
5.times { |i|
  puts i
}

控制權 1#

後面 block (你好) 沒有特別呼叫，不會動作

def say_hello
  puts "哈囉"
end

say_hello {
  puts "你好"
}

puts "安妮哈搜呦"

印出順序

哈囉
安妮哈搜呦

控制權 2#

透過 yield 呼叫後面的 block

def say_hello
  puts "哈囉"
  yield
  puts "空妮基挖"
end

say_hello {
  puts "你好"
}

puts "安妮哈搜呦"

印出順序

哈囉
你好
空妮基挖
安妮哈搜呦

控制權 3#

yield 可以帶入值，傳給 block 使用，而 block 也可以回傳最後結果

def say_hello
  puts "哈囉"
  calc = yield 3, 5
  puts "空妮基挖 #{calc}"
end

say_hello { |i, j|
  puts "你好 #{i} #{j}"
  i + j
}

puts "安妮哈搜呦"

印出順序

哈囉
你好 3 5
空妮基挖 8
安妮哈搜呦

控制權 4#

可以透過 « 將值傳送給變數

def select_odd (list)
  result = []
  list.each do |n|
    result << n if yield(n)
  end
  result
end

# 印出 [1, 3]
p select_odd([1, 2, 3, 4]) { |i|
  i.odd?
}

控制權 5#

可以透過 block_given? 去判斷是否有 block，如果沒有判斷，也沒有 block 卻用 yield 呼叫會報錯

def say_hello
  yield if block_given?
end

# 什麼都沒有
say_hello 

# 印出 hello
say_hello {
  puts "hello"
}

控制權 6#

{ } 與 do … end 其實還是有差別的， { } 的結合率較強，執行優先度會較大

list = [1, 2, 3, 4, 5]

# 印出 [2, 4, 6, 8, 10]
p list.map { |item|
  item * 2
}

# 印出 #<Enumerator: [1, 2, 3, 4, 5]:map>
p list.map do |item|
  item * 2
end

Proc 與 Lambda#

注意這些寫法

add_two_proc = Proc.new { |n| n + 2}
p add_two_proc.call(1, 2, 3) # 印出 3 ，後面參數被忽略
p add_two_proc.call(1) # 印出 3 
p add_two_proc[1] # 印出 3 
p add_two_proc.(1) # 印出 3 
p add_two_proc.===(1) # 印出 3 

add_two_lambda = lambda { |n| n + 2}
add_two_lambda2 = -> (n) { n + 2}
p add_two_lambda.call(1) # 印出 3
p add_two_lambda.call(1, 2, 3) # 錯誤，引數數量錯誤
